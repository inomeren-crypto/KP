2. ПАСПОРТ СИСТЕМЫ БЕЗОПАСНОСТИ

2.1. Общие сведения  
- Объект: Юридическая фирма в бизнес-центре  
- Количество сотрудников: 40  
- Дата ввода в эксплуатацию: — (проектная стадия)  
- Основание: Техническое задание (вариант 16)

2.2. Назначение и функции  
Система предназначена для:  
- Контроля доступа в зоны с конфиденциальной информацией (архив, переговорная);  
- Фиксации событий в зонах общего пользования (вход, коридоры);  
- Обеспечения соответствия ФЗ-152 «О персональных данных»;  
- Предотвращения несанкционированного доступа.

2.3. Состав системы  
- СВН: 3 IP-камеры Axis M3045-V  
- СКУД: 6 считывателей HID Signo 2k, 1 контроллер HID VertX EVO  
- Программное обеспечение: Milestone XProtect Express, HID Velocity Standard  
- Сетевая инфраструктура: VLAN 100, PoE+-коммутатор

2.4. Технические характеристики  
- Время хранения видеоархива: 30 суток  
- Разрешение камер: Full HD (1920×1080), WDR  
- Хранилище: 2×2 ТБ WD Purple в RAID-1  
- Сетевая изоляция: VLAN 100  
- Протоколы: ONVIF Profile T/C, OSDP Secure, RTSPS

2.5. Условия эксплуатации  
- Температура: +18…+25°C (внутри помещения)  
- Электропитание: ИБП на 2 часа для критических компонентов  
- Персонал: администратор с правами полного доступа

2.6. Ответственные лица  
- Администратор системы: —  
- Ответственный за ИБ: —  
- Оператор охраны: —  

2.7. Соответствие стандартам  
- ГОСТ Р 51558-2000 — технические требования к СВН  
- ISO/IEC 27001 — управление информационной безопасностью  
- ФЗ-152 — обработка персональных данных  
- ONVIF Profile T/C, OSDP — совместимость и безопасность  
- NDAA — оборудование от производителей Axis и HID Global

3. РЕГЛАМЕНТЫ И ИНСТРУКЦИИ

3.1. Регламент управления пользователями  
- Роли: Администратор, Оператор охраны, Сотрудник, Гость  
- Принцип наименьших привилегий:  
  • Оператор — только просмотр видео, без настройки  
  • Сотрудник — только физический доступ, без доступа к интерфейсу VMS  
- Процедуры:  
  • При приёме на работу — выдача карты в течение 1 дня  
  • При увольнении — отзыв доступа в течение 1 часа  
  • При утере карты — временная блокировка

3.2. Инструкция по резервному копированию  
- Что копировать:  
  • Конфигурация VMS (экспорт в XML)  
  • База СКУД (пользователи, права)  
  • Сетевые настройки  
  • Техническая документация  
- Как часто:  
  • После каждого изменения — обязательно  
  • Еженедельно — по расписанию  
- Где хранить:  
  • Внешний зашифрованный USB-носитель (не на сервере!)

3.3. Инструкция по восстановлению после сбоя  
1. Диагностика: определить, какой компонент не работает  
2. Остановить службу VMS/СКУД  
3. Заменить конфигурационные файлы из резервной копии  
4. Запустить службу  
5. Проверить подключение камер и считывателей  
6. Зафиксировать восстановление в журнале
