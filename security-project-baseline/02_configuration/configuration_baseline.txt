Проектная baseline и спецификация конфигурации  
Объект: Юридическая фирма (вариант 16)  
Дата формирования baseline: 04.12.2025  

1. СОСТАВ BASELINE  
Baseline представляет собой официально утверждённое состояние системы безопасности на момент завершения проектирования. Она включает:  
- Все схемы конфигурации (Deployment, Configuration, Integration);  
- Спецификацию оборудования и программного обеспечения;  
- Сетевые параметры;  
- Требования к интеграции подсистем.  

2. СПЕЦИФИКАЦИЯ ОБОРУДОВАНИЯ  
Оборудование выбрано в соответствии с принципами из Лекции №1 и требованиями ГОСТ Р 51558-2000, ONVIF, OSDP, NDAA.  

Система видеонаблюдения (СВН):  
- Количество камер: 3 шт. (внутренние, без уличных)  
- Модель: Axis M3045-V  
- Характеристики: Full HD (2 Мп), WDR, H.265, ONVIF Profile T, NDAA-compliant  
- Зоны наблюдения: холл (вход), коридор у архива, коридор у серверной  
- Запрет: видеонаблюдение не устанавливается в переговорных, архиве, кабинетах  

Система контроля и управления доступом (СКУД):  
- Количество считывателей: 6 шт.  
- Модель: HID Signo 2k  
- Тип идентификации:  
  • 5 зон — однофакторная (MIFARE DESFire EV3)  
  • Архив — двухфакторная (карта + биометрия)  
- Контроллер: HID VertX EVO с поддержкой OSDP Secure  
- Протокол: OSDP вместо Wiegand (требование безопасности самой системы)  

Серверная часть:  
- VMS: Milestone XProtect Express 2023 R2  
- ПО СКУД: HID Global Velocity Standard  
- ОС: Windows Server 2022  
- Хранилище: 2×2 ТБ WD Purple в RAID-1 (на 30 суток хранения)  

3. СЕТЕВЫЕ ПАРАМЕТРЫ  
- VLAN: 100 (выделена исключительно для СВН и СКУД)  
- IP-диапазон: 192.168.100.0/24  
- Сервер VMS/СКУД: 192.168.100.10  
- Камеры: 192.168.100.20–22  
- Считыватели: 192.168.100.40–45  
- Контроллер СКУД: 192.168.100.30  
- Коммутатор: HPE OfficeConnect 1950-24G-PoE+ (192.168.100.1)  
- Кабели: Cat 6 для всех линий  
- Питание: ИБП на 2 часа для критических компонентов  

4. ТРЕБОВАНИЯ К ИНТЕГРАЦИИ  
- Протоколы: ONVIF Profile C (СВН ↔ СКУД), OSDP Secure (считыватели ↔ контроллер), RTSPS (видеопоток)  
- Сценарии:  
  • При неуспешной попытке доступа в архив → автоматическая запись 30-сек видеофрагмента со всех 3 камер  
  • Успешный вход в архив → привязка события к ФИО пользователя в журнале VMS  
- Безопасность: все каналы зашифрованы (TLS 1.2+), принцип наименьших привилегий  

5. СООТВЕТСТВИЕ ПРИНЦИПАМ ИЗ ЛЕКЦИИ №1  
- Системность: все компоненты работают как единое целое  
- Надёжность: RAID-1, ИБП, резервное копирование  
- Масштабируемость: архитектура поддерживает расширение до 50 пользователей  
- Безопасность самой системы: VLAN, шифрование, NDAA-compliant оборудование  
- Комплексность: интеграция СВН и СКУД усиливает общий эффект безопасности  

Данная baseline зафиксирована в Git-репозитории и служит основой для всех последующих изменений в рамках процесса управления конфигурацией (RFC, аудит, восстановление).
